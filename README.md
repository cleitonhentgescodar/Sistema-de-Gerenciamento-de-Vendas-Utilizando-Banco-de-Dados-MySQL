# Sistema de Gerenciamento de Vendas utilizando MySQL

Este projeto consiste no desenvolvimento de um **banco de dados relacional utilizando MySQL**, criado como **iniciativa de aprendizado prático em Sistemas de Gerenciamento de Banco de Dados (SGBD)**.

O objetivo do projeto é aplicar conceitos fundamentais de **modelagem de dados, estruturação de bancos relacionais e consultas SQL**, simulando o funcionamento de um sistema de vendas de uma loja comercial.

Através da construção do banco de dados e da realização de consultas analíticas, o projeto demonstra como um SGBD pode ser utilizado para **organizar informações comerciais e gerar análises úteis para a gestão de um negócio**.

---

# Introdução

Com o crescimento das atividades comerciais e o aumento da quantidade de dados gerados diariamente, torna-se essencial que as empresas utilizem sistemas eficientes para **armazenar, organizar e analisar suas informações**.

Os **Sistemas de Gerenciamento de Banco de Dados (SGBD)** permitem que os dados sejam armazenados de forma estruturada, garantindo maior organização, segurança e facilidade de acesso às informações.

Neste projeto foi utilizado o **MySQL**, juntamente com a linguagem **SQL (Structured Query Language)** para manipulação e análise dos dados.

O sistema desenvolvido simula o funcionamento de uma loja comercial, registrando informações sobre:

- clientes  
- produtos  
- vendedores  
- pedidos  
- itens de pedido  

Essas informações permitem realizar análises importantes sobre o desempenho comercial da loja.

---

# Problema de Negócio

Em muitos pequenos e médios negócios, o controle de vendas, clientes e produtos ainda é realizado de forma manual ou através de planilhas simples.

Esse tipo de controle pode gerar diversos problemas, como:

- dificuldade na organização das informações  
- erros no registro de dados  
- perda do histórico de vendas  
- dificuldade na análise do desempenho da empresa  

Sem um sistema estruturado de armazenamento de dados, torna-se difícil responder perguntas importantes para o negócio, como:

- Quais produtos são mais vendidos?  
- Quais clientes compram com maior frequência?  
- Qual vendedor possui melhor desempenho em vendas?  

Dessa forma, surge a necessidade de utilizar um **SGBD para armazenar e analisar informações de forma organizada e eficiente**.

---

# Objetivos do Projeto

O principal objetivo deste projeto é **desenvolver um banco de dados relacional utilizando MySQL como prática de aprendizado em SGBD**.

## Objetivos específicos

- Criar a estrutura de um banco de dados relacional para uma loja comercial  
- Desenvolver tabelas para armazenar informações sobre clientes, produtos, vendedores e pedidos  
- Estabelecer relacionamentos entre tabelas utilizando **chaves primárias e estrangeiras**  
- Inserir dados simulados para representar operações reais de vendas  
- Realizar consultas SQL para análise das informações  
- Demonstrar como um banco de dados pode auxiliar na organização e análise de informações comerciais  

---

# Tecnologias Utilizadas

## MySQL
Sistema de Gerenciamento de Banco de Dados Relacional utilizado para criar e gerenciar o banco de dados do sistema.

Utilizado para:

- criação do banco de dados  
- definição das tabelas  
- armazenamento dos dados  
- execução das consultas SQL  

---

## MySQL Workbench
Ferramenta gráfica utilizada para desenvolvimento e administração do banco de dados.

Utilizada para:

- criação e administração do banco  
- execução dos comandos SQL  
- inserção de dados nas tabelas  
- execução de consultas analíticas  
- visualização dos resultados  

---

## SQL (Structured Query Language)

Linguagem utilizada para manipular bancos de dados relacionais.

Comandos utilizados no projeto:

- `CREATE DATABASE`
- `CREATE TABLE`
- `INSERT INTO`
- `SELECT`
- `JOIN`
- `GROUP BY`
- `ORDER BY`
